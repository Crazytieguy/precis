## Observations

Patterns noticed during fixture inspections. When multiple observations point to the same issue, it's ready to implement.

- ~~**Many small files with identical patterns dominate budget**~~ (partially fixed: directory cohort penalty via `dir_cohort_files`/`dir_cohort_avg_symbols` — when many files in the same directory have groups of the same kind_category with low avg symbols per file, a diminishing-returns penalty is applied: `factor = 1 - (1 - 5/n) * uniformity` where uniformity scales inversely with avg symbols. sps_core artifacts reduced from 23 individual files to mod.rs listing, freeing budget for exec.rs/link.rs. d2ts operators reduced from full class+method+factory per file to names-only for most, freeing budget for README headings, multiset body, core types. mcphost UI styling helpers eliminated, freeing budget for models/session/openai methods. semver functions/ directory eliminated. - **Pure-documentation directories produce heading-only output** (mdbook_guide_src) — The `guide/src/` directory is entirely markdown documentation (~40 files). Precis output is a detailed table of contents (section headings from every file) with virtually no body content. The Explore agent reads actual content and understands what the documentation says — key workflows, concepts, extension mechanisms. The one place body content appears (README.md), it's CSS from a `<style>` block (lines 4-5) rather than the descriptive prose (lines 20-34). This is the documentation-specific version of "many cheap heading items crowd out valuable body items" — hundreds of headings across dozens of files each cost 1 line, while body content competes poorly. Compounds with the ARCHITECTURE.md and log observations.
- **Package metadata and config files consume budget in small libraries** (mitt, tomli, xlstm) — For a 1000-token budget on a single-file library, package.json (~22 lines), tsconfig.json (~6 lines), and README boilerplate headings (License, Contribute, Reporting Issues, Submitting PRs — ~8 lines) together consume ~50% of output. Meanwhile the `mitt()` function body — the actual on/off/emit implementations with key design decisions (`.slice()` copy before iteration, `>>> 0` bitwise trick, wildcard-after-specific ordering) — is truncated at `return {`. The Explore agent read the full 124-line file immediately and understood the implementation. For small libraries, the implementation IS the architecture; type signatures (well-covered by precis) show the "what" but the function bodies show the "how." Compounds with the general "many cheap items beat fewer valuable items" pattern — package.json section headings and README headings are each cheap, collectively outcompeting more expensive body-line items from the single source file. In tomli, `pyproject.toml` `[tool.*]` section headers (isort, tox, coverage, mypy overrides — 18 lines, ~9% of budget) plus supporting scripts (fuzzer, profiler, benchmark — ~13 lines) together consume ~15% of budget with zero architectural value, displacing parser function body content that would show the actual parsing algorithm. In xlstm, 3 conda environment YAML files (`environment_pt220cu121.yaml`, `environment_pt240cu124.yaml`, `environment_pt260cu126.yaml`) consume ~6 lines showing just `name: xlstm / channels: / dependencies:` — zero architectural value.

- **C typedef struct fields invisible** (bareiron) — The 4 `typedef struct { ... } Name;` declarations in `globals.h` (PlayerData, MobData, BlockChange, ChunkAnchor) show `typedef struct {` with no field details despite the Type body value fix. Union fields may use a different node type. Needs investigation in layout.rs body truncation logic.
- **C preprocessor defines consume budget in large single-file projects** (sqlite_vec) — ~~`sqlite-vec.c` has ~10 lines of platform-compatibility preprocessor defines (`#define UINT32_TYPE uint32_t` etc.) at the top of the file with zero architectural value~~ (fixed: `_TYPE` defines now filtered). Additionally, 10 `vec0_column_*_idx*` helper functions follow an obvious pattern (distance_idx, k_idx, is_vector, to_vector_idx, is_partition, to_partition_idx, is_auxiliary, to_auxiliary_idx, is_metadata, to_metadata_idx). - **Unsupported languages make multi-language codebases unintelligible** (soluna) — Soluna is a 2D game framework ("Sokol + Lua = Soluna") with a C rendering engine and a Lua scripting layer. Lua (`.lua`) is not a supported extension, so the entire high-level API (`src/lualib/` with ~12 Lua modules: main.lua, soluna.lua, text.lua, layout.lua) and the service architecture (`src/service/` with render.lua, loader.lua, start.lua, gamepad.lua, log.lua — the ltask-based multi-threaded service layer) are completely invisible. The Explore agent read these immediately and understood the multi-threaded service architecture, Lua↔C binding flow, and the actual user-facing API. Meanwhile, precis output is dominated by ~140 lines of C struct definitions and `#define` macros — the low-level implementation details. The C code alone gives almost no hint that this is a Lua framework at all. Additionally, `luamods.c` (the central module registry listing all ~30 C↔Lua bindings) only shows two function names (`soluna_embed`, `soluna_openlibs`) without the module list that would reveal the framework's API surface. Crypto macros in `lcrypt.c` (GET_UINT32, DES_IP, DES_FP, DES_ROUND, LEFTROTATE — ~8 lines) and platform #defines in `entry.c` (~10 lines) consume budget on implementation details. Compounds with README body content being invisible (the tagline "A framework you can use to make 2D games in Lua with multithreading" would have been the single most useful line).

## Implementation notes

- **Doc comment detection** is partially migrated to tree-sitter. `Symbol.doc_start_line` is computed from AST sibling navigation, with text heuristic fallback. Handles `///`, `/**` (Rust), `/**` JSDoc (TypeScript), `//`/`/*` (Go, C), `#` (Python). Also handles wrapper nodes (`export_statement`, `decorated_definition`) for exported/decorated symbols. Python docstrings remain text-based heuristics. (Multi-line signature detection was also migrated to tree-sitter — `Symbol.sig_end_line` is computed from the AST body/block child, with text heuristic fallback for nodes without detectable body like type aliases and method signatures.)
- **Overload dedup** — consecutive function symbols with the same name are collapsed (keeping the last). Skipped for Go (`init()` functions).
